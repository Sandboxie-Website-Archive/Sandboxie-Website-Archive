Post by wraithdu » Sat Jan 05, 2008 11:05 pm

You are correct to use %LaunchDir%, ie

FileRootPath=%LaunchDir%\DATA\SANDBOX\%USER%\%SANDBOX%

What file copy errors are you receiving exactly? This could be something to do with permissions on your USB drive, or the app folder in particular. I doubt it's anything to do with the portable launcher. Do you get errors if you use a complete path, ie

FileRootPath=E:\SandboxiePortable\DATA\SANDBOX\%USER%\%SANDBOX%

where E:\ is the drive letter of your USB drive? You'll have to check the drive letter and change it in you INI file before running Sandboxie.

PLEASE READ THE README.TXT DURING INSTALLATION FOR INSTRUCTIONS

** DO NOT USE on a system with a locally installed version of Sandboxie!!


<-- NOW packaged as an installer!! -->

SandboxiePortable now uses the official Sandboxie installer. This has several benefits. Installation is now much simpler, you don't have to manually extract anything. Since the installer cleans up everything upon removal, you can now use the shell / sendto extensions portably.

I've also included an Updater utility to simplify updating to a new version of Sandboxie. It is located in the same directory as the launcher. A separate download link is below for users who do not need to update the whole package. Place the Updater in the same directory as the launcher.


** NOTE TO USERS UPGRADING FROM 3.01.21 or 3.02 -

The new GUI versions of Sandboxie use a new layout for the INI file. DO NOT use your old INI file directly with this new portable version of Sandboxie. Run the launcher once and let it copy a default INI to the application directory, then modify your settings via the GUI.

LINKS:

SandboxiePortable_2.3.0.4

SandboxiePortable Updater

- Installation and removal of Sandboxie now handled by official Sandboxie installer

- CrashCleanup utility updated accordingly

- Sandbox location now read from FileRootPath setting in the GlobalSettings section for compatibility with new Sandboxie behavior

*** WARNING ***
