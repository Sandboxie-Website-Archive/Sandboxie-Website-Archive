Unexpected behavior when copying to current directory.

Posted: Mon Jun 13, 2016 12:56 pm
by JL6
I am using Sandboxie 5.10 on Windows 7 (64-bit). Apologies if this issue has been addressed somewhere and I didn't see it.

First, to illustrate normal Windows behavior:
1. Open unsandboxed explorer, make a new folder, and make a new text document inside that folder.
2. Right-click on the file, choose copy, then right-click in the window and choose paste.
Result: Windows makes a copy of the file called "file - Copy.txt". Repeat right-click and paste will create "file - Copy (2).txt", etc.
3. Go up a directory, right-click on the folder, choose copy, then right-click in the window and choose paste.
Result: Windows makes a copy of the folder called "folder - Copy". Repeat right-click and paste will create "folder - Copy (2)", etc.

In SBIE:
1. Open sandboxed explorer, make a new folder, and make a new text document inside that folder.
2. Right-click on the file, choose copy, then right-click in the window and choose paste.
Result: Nothing. One way around this is to copy the file somewhere else, then copy that file back to the original folder.
3. Go up a directory, right-click on the folder, choose copy, then right-click in the window and choose paste.
Result: Windows says the destination folder is the same as the source folder, and the only option is to cancel. One way around this is to copy the folder somewhere else, rename the folder, and then copy it back.

Thanks for looking into this.
