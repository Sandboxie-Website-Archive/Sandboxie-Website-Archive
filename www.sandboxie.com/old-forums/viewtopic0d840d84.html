Post by bo.elam » Sun Mar 31, 2013 9:41 pm

For your Firefox sandbox, don't Force, C:\WINDOWS\System32\Macromed\Flash, flashplayerplugin_11_6_602_180.exe , plugin-container.exe. Firefox is the only program that you need to force in your Firefox sandbox. No need to force a folder neither.

I believe you are using/setting up a restricted sandbox. In Sandbox settings for that sandbox, in addition to Firefox, you need to allow Plugin container and flashplayerplugin_11_6_602_180.exe permission to Start and Run.

Sandbox settings>Restrictions>Start/Run access.

Bo

Drew99GT wrote:I have a question: if you have the flashplayer protected mode disabled (which greatly improves performance in Firefox), will flashplayer automatically start in when Firefox is opened in a sandbox, or do I need to add a command to start flashplayer sandboxed?

Thanks!
In an unrestricted sandbox, you dont need to add nothing. Plugin container/Flash will start and run when you visit a site with Flash content. If your Firefox sandbox is restricted, in XP, you need to allow Plugin container Start/Run access. In W7, (with the latest Flash version) you must allow Plugin container and Flashplayerplugin_11_7_700_169.exe in order for Flash to run when you visit a site like Youtube.
