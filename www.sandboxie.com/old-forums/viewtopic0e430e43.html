Hi plastic00,

The first screenshot shows the Direct Access paths that are already granted to ChromePrograms (they are the default paths to allow Chrome access to the phishing Database). In order to add something else, you must select the Program (for example <ChromePrograms>, then click "Add" and add the path you want the program to have direct access to.
After that, you will be able to hit "Apply".
As for the second screenshot, there is nothing added by default to "chrome.exe" itself, so you you will have to click "Add", and select the path (s) you want it to have Full Access to.
For more info, check out this entry:

by Syrinx
Sphinx, do you by chance use the BlockProcessAccess Addon aka SBIExtra.dll?

This one stood out in your log

Code: Select all

Ipc      X  $:chrome.exe
If chrome is only running inside of sandboxie I don't see why it should be prevented from writing to a same-box instance of itself.

For troubleshooting purposes only you could try adding an OpenIPCPath=chrome.exe,$:chrome.exe under Sandbox Settings > Resource Access > IPC Access > Direct Access
Add Program > chrome.exe
Add > $:chrome.exe

Just be aware that this will still allow a sandboxed version of chrome to write to an unsandboxed chrome or even across boxes if you run more than one instance at a time. For testing the crash this might be good but I would not suggest keeping it for actual everyday usage even if it does aid in stopping the crash.
