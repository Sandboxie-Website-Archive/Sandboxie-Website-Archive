Post by Barb@Invincea » Tue Oct 10, 2017 2:33 pm

Hello Mark_Tasso2 ,

I tested Spotify Web and was unable to hear sounds via Chrome or Firefox [Win 10 x64 + sbie latest beta 5.21.4 - No antivirus].
I was able to get it to work with Chrome by opening a hole in the Sandbox:
Right-Click on your Sandbox --> Sandbox settings--> Resource Access --> COM Access
Hit Add and paste
{77A1C827-FCD2-4689-8915-9D613CC5FA3E}
Hit Apply and OK your way out.
Configure--> Reload Configuration
Re-try Spotify web after deleting the contents of your Sandbox.
I have not tested this COM class extensively, so I am not sure if it will cause any undesired behavior. I recommend that, if you really need to run Spotify Sandboxed you either run the client, or that you create a Sandbox that only allows Chrome and apply the workaround.

The client did not present any issues for me.
Here's how I tested it:
I downloaded Spotify 1.0.64, installed it on the host and tested it Sandboxed ---> Able to play songs from the "home page" (under "Afternoon Soundtracks" ...Default view of Spotify Free , no customization added).
I updated it (outside Sandboxie) to 1.0.65, retried using it Sandboxed and audio played just fine.

Can you provide more information regarding how are you reproducing the problem?
