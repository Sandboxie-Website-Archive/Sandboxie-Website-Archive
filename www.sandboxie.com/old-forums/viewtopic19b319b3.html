I recently posted the same problem. After more research, I found an answer that worked: http://forums.sandboxie.com/phpBB3/view ... ve#p120679

"On a site that implements WebRTC, Chromium and Chromium base browsers are using the Quality Windows Audio/Video Experience(QWAVE) API to set DSCP bits on outgoing UDP Packets. This requires access to \Device\QWAVEdrv. So Chromium browser is going to crash on any site that implement it and NOT the SbieCtrl.exe"......

Hope it helps.
