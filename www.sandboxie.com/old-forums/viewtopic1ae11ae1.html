Barb,

I figured out what is really going on with my problem. I had set the cache location of IE11 to my ramdisk drive Z. However, I had changed the drive letter when I was testing some software and reverted back later, IE11 changed its cache folder into the sandbox which is weird. The counters2.dat file belongs to IE11. After I moved its cache folder to another location and everything is back to normal again. I can finally clear the sandbox.

As for Chrome constant CPU usage, it is Chrome's 58 problem, not Sandboxie. I open Chrome and go to google.com and let it idling while I check out the background process in Task Manager and Resource Monitor, I found Chrome keep using CPU for doing nothing. IE11 and Edge doesn't do that. Google needs to fix this problem as it is draining laptop battery life when sitting idle.
