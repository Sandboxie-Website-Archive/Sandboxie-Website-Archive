by _is_m00nbl00d
Hello aigle,

I don't think there's any need to complicate the simplicity behind Sandboxie. I also don't see why such would be needed.

If you sandbox the dangerous applications (web browser, e-mail client, media player, image viewer, messenger, etc), then everything else will also become sandboxed.

If you'd like to run an application unsandboxed, e.g web browser, but would still like to have other applications forced, such as PDF reader when you open a PDF file from within the web browser, then you could subscribe to my feature suggestion here http://www.sandboxie.com/phpbb/viewtopic.php?t=10281

As it is right now, even if you have e.g. PDF reader forced to its own sandbox, it will run unsandboxed. It would be nice to have it forced to its sandbox, if initiated by an unsandboxed process.

Would this approach be something within what you'd like Sandboxie to offer? Otherwise, what you're suggesting would make of Sandboxie a more complicated application, IMHO.

Then again, only Tzuk can say for sure whether or not something that would play nice.
