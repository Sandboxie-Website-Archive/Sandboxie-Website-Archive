Post by Syrinx » Tue Sep 05, 2017 7:44 pm

I can't reproduce it on a clean install of Windows 10 CU with Sandboxie 5.20. As a workaround I'd try going into your boxes and removing whatever rule exists there that includes %temp% or set it up with the actual path if you're the only user.

The odd part is that it isn't actually having the problem with %temp% itself but it can't resolve the %USERPROFILE% part so it'll likely effect all other similar variables or anything else that uses it.

Another option might be to try setting up a new user account and see if it happens with that as well.

If you're comfortable with regedit you could also navigate to HKEY_CURRENT_USER\Volatile Environment and see what path is set for USERPROFILE and if it's accurate. It should be C:\Users\*YOURUSERNAME* by default
