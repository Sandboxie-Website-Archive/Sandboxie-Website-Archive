Post by Syrinx » Sun Sep 03, 2017 12:49 pm

I don't usually mess with the beta/nightly builds but a quick google search says pingsender is used for sending telemetry back. I'd try adding firefox.exe as a leader program and pingsender.exe as a lingering program so hopefully that when firefox terminates SBIE will kill pingsender and the box will close/delete as seems to be your desire.
Sandbox Settings > Program Stop > Lingering Programs/Leader Programs

If there is anything left running in the box Sandboxie will not try to preform the close actions. If the box closes but Sandboxie can't delete it (usually due to file access issues with another program outside the box) then it should pop up an error informing you, are you getting those?

OK - here are some test results...

With "Pingsender.exe" blocked - the sandbox contents are NOT deleted, but there are NO lingering programs - this happens 100% of the time & there is a popup of "pingsender is canceled..." if the popup box is selected in settings - but there is no mention of the contents not being deleted. So pingsender.exe definitely seems to be causing problems for Sbie
