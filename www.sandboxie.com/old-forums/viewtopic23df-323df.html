You can try to use the SDelete command instead of RMDIR and see if the errors remain.

On the other hand the error message you have provided, says :"The system cannot find the file specified"

It seems that the RMDIR command is unable to find the file to delete. I do not know for sure but It may have something to do with the 8dot3 naming in windows 10. I believe the 8dot3 file naming is not enabled by default in windows 10. You can query the setting in a cmd window (with admin rights) and the command: fsutil behavior query disable8dot3 c:
when I execute this command, I get:

The volume state is: 0 (8dot3 name creation is enabled).
The registry state is: 2 (Per volume setting - the default).

Based on the above two settings, 8dot3 name creation is enabled on c:

If it turn out that 8dot3 is disabled, you can try to enable 8dot3 and see if this fixes your error.
Please read up on the subject. I do not know for sure If this is the issue. Please backup your system before making any changes.
