I can't say I've seen "empty" folders laying around in the SB. nVidia, which I'm assuming would be driver type programs, shouldn't be installed directly into the SB anyway. To delete anything in the SB, one should delete the contents of each sandbox. This is the correct way of doing it.

The sandbox isn't meant to be a permanent storage OR a repository for running programs.
It's fine to install and test (however, that's becoming more and more less frequent with Windows 10)
....but the purpose of the SB is to run a program under the safety of SBIE and then delete the contents...As SBIE is not a VM.

The hierarchy it creates is rudimentary and only for a scratch space when running browsers, files, etc in that temporary container.

If you have a program you use frequently, and you deem it safe. Install it on your host. If you gone on line with that program, run it "as sandboxed" and routinely delete the contents of your sandbox.
