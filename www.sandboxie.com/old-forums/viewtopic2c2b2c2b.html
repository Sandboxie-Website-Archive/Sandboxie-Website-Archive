Hello yodar444,
what does [--] mean?
It means it is added globally and you will need to use the Software Compatibility tool to remove it (Configure --> Software compatibility).
why is LastPass listed here?
It was probably detected by Sandboxie in the past and added to your global templates.

I installed LastPass on Firefox and then launched it Sandboxed. LastPass appeared just fine. Please try a new Sandbox with default settings to see if that solves the problem.
You can also try removing the lastpass template as explained above, to see if that makes a difference.

If the issue persists, please try the latest beta:

what does [--] mean?
why is LastPass listed here?
[--] means that the setting comes from a template, as a result of having it listed under [GlobalSettings] in the Sandboxie configuration file (Sandboxie.ini):

[GlobalSettings]
...
Template=LastPass
...

In other words, that is normal and correct for all sandboxes, since Sandboxie detected that LastPass is being used.
My LastPass icon is at the top, next to the address bar. But I use Cyberfox, so I don't know of any problems with Firefox.
I assume that you have Deleted Contents of the sandbox, since that will force Firefox to re-read the profile files from unsandboxed Firefox the next time that it runs sandboxed.
Also, you might try using "Customize" when not sandboxed, move the icon from the add-on bar so that it's next to the address bar, and make sure that you have Deleted Contents of the sandbox before running Firefox sandboxed, again.
