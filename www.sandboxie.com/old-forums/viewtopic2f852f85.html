Applications attempting to run in the sandbox they were downloaded in?

Posted: Fri Mar 09, 2018 4:39 pm
by famadeus
Recently I have noticed a bizarre issue where applications that have been downloaded in a sandbox, moved out of that sandbox and then either run in an unsandboxed environment or in a different sandbox altogether will attempt to load in the sandbox they were originally downloaded in, as if they are somehow retaining properties inherited from the sandbox.

This happened once using an emulator called 'Cemu' which also seemed to retain other properties in some manner, ie. once running in a new sandbox (not the one it was downloaded in) it was looking for directories where it was originally extracted, the Desktop for example, instead of the new directory where it was placed. I ignored this as a one off. However this has now happened again using 'madVR', a video renderer. When loading a video the video opened with MPC-HC video player but also loaded the madVR video renderer in a sandboxed environment (the sandbox it was originally downloaded in).

I am struggling to reproduce the exact steps that lead to this issue so I'd like to ask if it's a known issue, unknown issue or just a user error? I hope by asking I can properly reproduce the issue, learn how to avoid it in the future if I'm doing something wrong and file it as a proper bug report if required.

- The sandbox I downloaded the applications in has 'Drop Rights' activated.
- In both cases the applications were cut from the sandbox and pasted to Desktop and then moved to the required folders.
- I'm currently using 5.23.3 beta due to the WIndows 7 kernel changes.
- Windows 7 x64, happened with downloads via Firefox 56 + 58 (Quantum), 7zip being used to extract program archives.
