Post by Curt@invincea » Mon Apr 27, 2015 4:54 pm

4.17.3 is released.

The AllowPrintToFile is going to cause some confusion, because it is a very complicated problem. The print spooler security hole is blocked -- no one needs to do anything unless you start getting the "SBIE1319 Blocked request to print to file".

Would it be better to change this flag to "AllowStartDocPrinter"? That is more specific to what it actually does, but is a bit cryptic. It is probably confusing to see in Win 7 you can still print to files even though AllowPrintToFile=n.
