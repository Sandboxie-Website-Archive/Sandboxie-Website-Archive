MS2001 wrote:I cannot print unless i either include the Openfilepath lines or I restart Windows because if I dont include the Openfilepath lines then the Print queue locks up regardless of whether I ignore or double click the 1320 error message

Thanks
This can occur if your print drivers/apps are not able to handle a STATUS_ACCESS_DENIED coming from the print spooler. If the user has access to the folder, they expect the spooler does as well, so they may not have coded for the denial. Manually clearing your print queue will probably fix the problem without rebooting.

Do not mess with c:\windows. And I don't see a reason to OpenFilePath=firefox.exe,C:\Windows\dellstat.ini unless this file is written to every time you print (which would require elevation as well). Your other opens are fine.
