Posted: Thu Mar 07, 2013 3:35 pm
by tzuk
If you would like to go back to using "clcl", version 4.01.03 may fix the problem.

I say "may" because it turns out Sandboxie has to adjust the data that was copied into the clipboard. Usually this happens before "clcl" would get to inspect the data, so everything seem fine. But it is possible "clcl" would get to the data first, and see it before the data was adjusted by Sandboxie. Unfortunately there isn't a 100% solution in this case.

My apologies.
The problem was that installing the .13 beta changed the settings in Webroot WSA from allow to deny/monitor for the sbie processes.
I have changed them back to allow and the clipboard is working again.
