Post by Guest10 » Fri Dec 07, 2012 9:16 am

It looks like the template for saving files to the entire Opera profile folder may not have a setting that will work for saving of the favicon files.

Look for Opera's "icons" folder in the Opera profile folder that's outside of the sandbox, so you know its location.
Then add a direct access setting for opera.exe, so it can save files to the "icons" sub-folder.

Outside of the sandbox:
XP:
C:\Documents and Settings\(user)\Local Settings\Application Data\Opera\Opera\icons

Vista/Win7 should be:
C:\Users\(user)\AppData\Local\Opera\Opera\icons

Sandbox Settings > Resource Access > File Access > Direct Access
"Add Program" button: opera.exe
"Add" button: Navigate to the "icons" folder. Select it.
OK your way out.

Tested on Opera 12.11

Looking for the "icons" folder meant to run Windows Explorer and use it to navigate through the folders and files on the hard disk.
Using the Sandbox Settings step, as I listed above, amounts to doing the same thing - but using Windows "Browse" windows to navigate thru the hard drive to the "icons" folder.

If you want to assume that your "icons" folder is in the default location, then a procedure that you can use is:

Sandbox Settings > Resource Access > File Access > Direct Access
"Add Program" button: opera.exe
"Edit/Add" button: %Local AppData%\Opera\Opera\icons\
OK your way out.

(note the <space> in: %Local AppData%, above)
