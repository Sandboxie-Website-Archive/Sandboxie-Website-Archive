Dun wrote: Firefox 32.0.3 doesn't start at all, doesn't matter if dropped rights are ticked or not. An error comes up: "FF is already running but not responding".
Perhaps testing previous Sandboxie betas has left garbage in your Sandboxie sandbox used by Firefox or corrupted your real Firefox profile. Hopefully, such problems are cured by:
1. deleting Sandboxie's sandbox
2. deleting the parent.lock file in your Firefox profile
3. creating a new Firefox profile
See https://support.mozilla.org/en-US/kb/fi ... responding

If you can you start Firefox unsandboxed without problems, it would provide evidence that there is a problem with the Sandboxie version you are testing.

Thanks for helping to ensure that the released version of Sandboxie is stable with Firefox!

Clipboard01.png (8.5 KiB) Viewed 4200 times
I also had another weird error:
SBIE1241 Cannot mount registry hive: [C0000043 / 22]
SBIE2314 Canceling process chrome.exe

I had to change some things around to get rid of both those errors. What I did was I went in my sandboxes and deleted the drive directory, so for example
C:\Sandbox\Username\Chrome\drive
because I've noticed what happens is sometimes chrome will try to update itself in the sandbox. in my case I found a chrome.exe and new_chrome.exe in the sandbox. Anyway I wiped out the whole drive part. If someone is going to try that know it wipes out everything sandboxed in the drive. You could probably just wipe out google part, like
C:\Sandbox\Username\Chrome\drive\C\Program Files (x86)\Google
In either case user settings for chrome are stored in the user not the drive folder.
Now I have upgraded outside of sandboxie by downloading latest x86 chrome standalone installer for all users (38.0.2125.104) and have no errors... so far...
