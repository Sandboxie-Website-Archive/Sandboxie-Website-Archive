Find the latest portable version here and use Eraserl.exe and Eraser.dll from that package - http://bbs.heidi.ie/viewtopic.php?f=14&t=5341

The newer normal buillds use the VC++ runtimes and are not portable, hence the statically linked portable build on that page.

After that, follow the readme instructions -

----------------------------
PORTABLE INI FILE OPTIONS:
----------------------------

To use any INI file options, copy 'SandboxiePortable.ini' from the
'Other' folder to the same folder as 'SandboxiePortable.exe'.


---------------------------------
USE ERASER TO WIPE THE SANDBOX:
---------------------------------

Optional settings -

EraserMethod=<choose a method>
EraserOption=<-silent|-results|-resultsonerror>

....etc.

Post by wraithdu » Mon Jan 12, 2009 12:29 pm

No, not at all. The portable launcher does not shutdown Sandboxie at all. It waits for Sandboxie Control to close, then invokes SB's own uninstall routine.

Anyway, a process running outside of Sandboxie will be able to close Sandboxie and it's service, provided you have sufficient privileges (Admin rights). This is not a security hole, risk, or concern. Sandboxie is meant to control processes running Sandboxed. Such processes cannot unload Sandboxie in any way which would cause a security leak.
