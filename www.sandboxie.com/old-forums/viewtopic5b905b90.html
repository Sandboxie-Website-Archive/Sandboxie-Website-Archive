I tried this on 64-bit Windows 7 (but not Windows 8 or XP) and I could reproduce a crash during printing so I assume it is the same crash.

I fixed it by adding compatibility settings for SolidConverterPDF, which would be offered to you automatically on upgrade to version 4.10.12.

Please note that the PDF viewer part of Solid Converter seems to hang under Sandboxie, so I added a setting to block it from running under Sandboxie. You'll get an error that Sandboxie is canceling some process, you can safely hide it and ignore it. (I.e. just click the HIDE button when the message appears the first time.)
