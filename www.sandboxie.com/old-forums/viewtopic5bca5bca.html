I noticed bsa.conf contains next lines:

Code: Select all

[BSA]

InjectDll=C:\Program Files\Sandboxie\Buster Sandbox Analyzer\sbiextra.dll
InjectDll=C:\Program Files\Sandboxie\Buster Sandbox Analyzer\antidel.dll
InjectDll=C:\Program Files\Sandboxie\Buster Sandbox Analyzer\LAPI.dll
OpenWinClass=TFormBSA
ConfigLevel=6
Enabled=y
NotifyInternetAccessDenied=y
BoxNameTitle=n
BorderColor=#0000FF

OpenFilePath=C:\Sandbox\TEST!!!\
"NotifyDirectDiskAccess=y" is missing. Without that parameter, BSA will be unable to notice direct disk writing behaviour.

Other two important parameters are:

ProcessLimit1=20
ProcessLimit2=30

That is used to avoid certain type of malware that floods the system making it unstable. Normally none (malicious or not) application will be running over 30 processes at the same time.

More things...

You include "DLLLDR.EXE". That file is obsolete and can be removed because it is not used anymore.

The same happens with "HideDriver.sys" and "HideDriverGUI.exe". They were replaced by BSA.SYS. Anyway someone could use HideDriver instead BSA´s own driver, so I could understand why it is kept in the package.

"regdump.exe" is dropped everytime BSA needs it, so it is unnecesary to have it in the package.

The same for "sigcheck.exe" and "ssdeep.exe".
