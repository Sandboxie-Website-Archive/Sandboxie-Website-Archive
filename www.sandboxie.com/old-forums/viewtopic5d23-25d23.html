I'd like to use Sandboxie at work (I've already talked to tzuk about licensing costs), but the lack of network drive support is pretty much a show-stopper. The main user application I'd like to sandbox is a network app -- it runs off of the network and writes to the network. I have no way of changing this.

So, what are the chances of network drive support being added into a future release of Sandboxie?


Barring network drive support, is there any way to sandbox *just the child processes of a given application*? That is, don't sandbox the main application (since it doesn't work when sandboxed), but automatically sandbox anything it spawns? While not perfect, I believe this would be a satisfactory work-around.

Thanks!

Posted: Thu Jan 11, 2007 12:57 pm
by tzuk
In version 2.7x, there is no special treatment for network drives. They get sandboxed just like any local drive. This means you can apply the usual OpenFilePath also for network drives.

In other words, you could say: these drives (or maybe just particular folders or files within them) should not be sandboxed.

I think that would solve your problem.
