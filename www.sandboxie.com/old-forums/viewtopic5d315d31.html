Post by BUCKAROO » Sat Jun 06, 2015 10:58 am

DocMAX wrote:... which OpenPipes i have to choose to let streaming work?
So I had to lug the laptop out of the cupboard for this. And . . . back in the cupboard it goes.

Host and client were Windows 8.1 + fwiw latest Apple Boot Camp Software Graphics Driver from AMD on the server end.

notes:
SBIE2101 Object name not found: \Device\QWAVEdrv, error CreateFile (C0000022) access = 00120089
SBIE2314 Canceling process Steam.exe
(On the host/server computer, apply the OpenPipePath setting and restart Steam.)

JustCause2.exe goes missing during and after every play...
(Delete the 0 byte JustCause2.exe in Sandbox contents and apply the ReadFilePath setting.)

Not a Sbie issue but if In-Home Streaming is not receiving broadcast message from clients, then, with Steam running, disconnect and reconnect to your LAN or WiFi.

Tested a few games, seems extra jittery. May be Sandboxie placebo effect?

[Steam]
Enabled=y
ConfigLevel=7
Template=BlockPorts
Template=LingerPrograms
BorderColor=#808080
OpenPipePath=\Device\QWAVEdrv
ReadFilePath=*\JustCause2.exe

Was plain to see from the Messages from Sandboxie popup. Import this into your registry if you want all SBIE messages even those marked as hidden to be logged to file. Though it showed up in my Resource Access Monitor as well with a X. I don't know. Lucky.

Code: Select all

REGEDIT4

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\SbieSvc]
"LogFile"="1;C:\\SbieLog.txt"
