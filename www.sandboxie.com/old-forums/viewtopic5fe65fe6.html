Another thing to mention, when updating nvidia driver. It will fail if there is sandboxed process being run due to sandboxie locking system files. If I remember correctly the catroot database file by sandboxie crypto/bits process which windows service try to access - this happen from previous sandboxie version not sure if it happen on latest one. So a tip here, if some apps doesn't start correctly, try close all sandboxed process first.
  1. It is not "some random number", those are important error codes.
  2. Apps do not "leak" in or out of Sandboxie. They are either started in the sandbox or out of the sandbox, depending on the configuration.
  3. Sandboxie does not lock system files. The sandboxed application may be doing that. But, that would be the same if the app were running outside Sbie.
  4. Are you trying to run VS in Sbie? I can't think of any reason why msbuild would start in Sbie unless you are running VS in Sbie or you have used a forced folder.
