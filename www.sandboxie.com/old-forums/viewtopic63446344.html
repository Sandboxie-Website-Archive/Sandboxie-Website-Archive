Looks like F-Secure maybe added one more resource. AndromedaXS please try:

Open Sandboxie Control, go to Sandbox Settings, expand Resource Access, IPC Access, Direct Access, click Add, paste:

*\BaseNamedObjects*\pipeservertaken*

Then click OK all the way back and restart your Firefox under Sandboxie. If it still doesn't work, I would like to see a new resource log, with this setting still in effect.

As you can see some of these numers are higher than what you posted. Maybe another update was released and it fixes the problem?

* * *

EDIT. I was checking the above with Sandboxie version 4.01. I checked again with Sandboxie 3.76 and Firefox doesn't even start.

But, I opened Sandboxie Control > Configure menu > Software Compatibility and disabled compatibility with F-Secure, then Firefox is working normally again.

So as a first step I suggest that you also try to disable compatibility with F-Secure in the Software Compatibility window in Sandboxie.
