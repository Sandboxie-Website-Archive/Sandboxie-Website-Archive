I've fixed this by uninstalling Bitdefenter Internet Security 2015. It has too much incompatibilities with software. Even adding Sandboxie processes to BIS active anti-virust control whitelist has not solved this.

Chrome crashes on 4.13.5 beta with dropped rights and gives an "Memory could not be read" error like this: http://forums.sandboxie.com/phpBB3/down ... .php?id=94 . Using 4.13.5 alpha there is a problem with "SandboxieDll.dll failed".
I finally find the way to get Chrome x64 and dropped rights working on Win7 x64. Just make clean install 4.13.5 beta, leave default configuration, close SB control and disable SB service then install 4.13.5 alpha on the top. I did that way few days ago and haven't seen Chrome crashes and "SandboxieDll.dll failed" errors. Adobe Flash Player 15,0,0,152 NPAPI installed inside Chrome sandbox also works.
