Post by wraithdu » Tue Sep 25, 2007 8:57 am

Could you recheck your steps in creating the portable version by going through the README again? It sounds like the driver and service are not being installed. The only reason I can think for this to happen is that the directory structure has been created incorrectly, or (more likely) the kmdutil.exe utility is not present in the Sandboxie directory along with the program files. If it is missing, it will fail silently to install/start the driver service.

If all the above is correct, you can take a screenshot of the program directory. Also, when the portable version is started, is the Sandboxie icon in the sys tray yellow like normal, or gray with a red dot in it? If it's gray, at a cmd prompt type

sc query SbieDrv
and
sc query SbieSvc

to verify the driver/service installation.
