Posted: Wed Feb 07, 2018 6:14 pm

Today I was using Chrome in the sandbox. 10 minutes later, web pages wouldn't load. I exited Chrome and tried to open it again, but nothing happened. I tried multiple times with no success. Chrome is not running in Task Manager. One minute it worked and suddenly nothing works.

I'm on Windows 10 Home 64-bit and using Bitdefender 2018 (paid version). I did not turn off protection in Bitdefender to make browsers work in the sandbox; after adding the following to the config file everything worked fine: WriteFilePath=C:\Program Files\Bitdefender\*. I have been running in this configuration for months with no problems.

The shortcut link to launching Chrome is "C:\Program Files\Sandboxie\Start.exe" /box:DefaultBox C:\Program Files (x86)\Google\Chrome\Application\chrome.exe --ssl-version-min=tls1. Chrome version 64.0.3282.140, Sandboxie beta version 5.23.5.
