OK, more research done.

Apparently, MT4 doesn't even need to be installed! I tried to simply copy/paste the directory with the old installation that no longer works in Sandboxie from the Program Files (86) in the sandbox to just a folder on my data drive (D:\test) on the physical machine on which it has never been installed before. Then I copy/pasted it into a fresh XP machine in Virtual Box. And finally to a humble USB stick.

In all these cases, that old installation that no longer works in Sandboxie woke up and ran fine - without any installation. It just asked for the login information.

However, when I tried to create a fresh Sandbox, open sanboxed Explorer and copy/paste the working installation into the new sandbox, it immediately crashes upon start again.

CONCLUSION: Since last week:
1. Existing MT4 installation no longer works in Sandboxie
2. New installation of MT4 doesn't work in Sandboxie
3. MT4 works perfectly, even without installation, in any folder on any drive, including removable ones, on physical Win 7 Pro 64-bit machine
4. MT4 works perfectly, even without any installation, in any folder on any drive on Win XP Pro 32-bit machine in Virtual Box
5. Any of the working MT4 copies immediately stop working again once copied back into Sandboxie.
6. This behavior started after about April 8, 2014 sometime after Windows Update
7. Unfortunately, I deleted my System Restore point, so I can't verify that MT4 would start working in Sandboxie again, if last weeks Windows Update would be reversed, which in turn makes me unable to verify that the Windows Update is indeed the culprit.

PiwPi, if you or someone else still have the System Restore Point and are willing to test the return before last week's Windows Update, please report your results here.
