If you have changed the installation path of Sandboxie, the default template "OfficeClickToRun" won't work. I'm not sure, but I think that just copying a template from Templates.ini to Sandboxie.ini doesn't work also. (Usually you have to rename templates and change their class to "Local".)

So, you could try the following:
- Remove both templates from your Sandboxie.ini.
- Add the following line to the settings for the box [Outlook]:

Code: Select all

OpenFilePath=<Office_Outlook>,%userprofile%\outlook files
- Save the changes.
- Run Notepad as Admin, open the file Templates.ini and search for [Template_OfficeClickToRun].
- Correct the path for "HostInjectDll=" and "HostInjectDll64=":

Code: Select all

HostInjectDll=C:\Program Files\Sec\Sandboxie\SboxHostDll.dll
HostInjectDll64=C:\Program Files\Sec\Sandboxie\SboxHostDll.dll
- Save the changes and reboot Windows.

Unfortunately, you have to do the Templates.ini corrections after every version update of Sandboxie! (I hope the devs will include a better solution soon.)

I didn't have auto delete set on outlook sandbox and accidentally deleted the contents (outlook&personal.pst)
You may want to give direct access to your outlook pst, otherwise it will take a long time for it to be recreated.

Create a new Sandbox and add Outlook to Sandbox Settings ---> Applications ---> Email Reader
Go to Appearance and check the Don't show Sandboxie indicator in the window title (to avoid freezing reminders).

Follow these steps to give access to your pst files' location:
https://www.sandboxie.com/index.php?App ... gs#folders

Then, use this tutorial (provided outlook launches this time), to test it: (Make sure Outlook works outside Sandboxie with the current profile prior to launching it sandboxed)
https://www.sandboxie.com/index.php?Tes ... figuration

If Outlook is still not launching, try disabling/remove any security software that might be installed on your computer (for testing purposes).
