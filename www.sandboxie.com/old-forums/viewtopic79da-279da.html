I’ll be repeating myself again. This is due to you or anyone using your laptop being unaware of using your Firefox unsandboxed. There are two theory I have. First is that you intentionally run the browser unsandboxed to make your browser up to date then being unaware and have use it to visit youtube.com(You are just human, you can’t just say that you can’t have this mistake). Second, SBIE service was disabled(for unknown reason), failed to run, or failed to load sbie driver. When Windows started and sbiectrl.exe detected the service isn't running then it will try to run the service. If you didn’t disable your UAC you’ll be prompted with Yes or No but if disabled it will be silent without you being notice. So when this happen you’ll be running Firefox unsandboxed, for some users, you or anyone might be unaware but you can notice this by looking the SBIE tray icon.

Hi Bellzemos. Thats a luge list of cookies. The only thing I can figure is that you ran Firefox unsandboxed for a while and then found the cookies when running CCleaner. I certainly never seen anything like that. I suggest you add Firefox to the Program alert list. That way, if for whatever reason Firefox runs unsandboxed, you ll get a SBIE message 1301 warning you about.
http://www.sandboxie.com/index.php?ConfigureMenu#alert

You could also create a new sandbox and use it to Force Firefox instead of the old DefaultBox. Another thing you can do is (something that I do) set cookies in Firefox Options to be kept until Firefox closes. Set "Keep until" to "I close Firefox". You can do this by using custom settings for history.
https://support.mozilla.org/en-US/kb/se ... -history_2
