Post by mooop12 » Tue Feb 02, 2016 4:03 pm

Found a fix. There seems to be an ongoing unreported Windows bug at least since Windows 7 where Visual C++ Redistributable package doesn't install properly and the system tries to load 32-bit libraries instead of 64-bit versions. This was the case here, manually extracting msvcr100.dll from the Visual C++ 2010 x64 package and putting it in Sandboxie directory fixes the problem.

**************************

For anyone else in the future having a similar problem, mooop12's solution worked, but I had to figure out how to do it by some trial-and-error and further web searching. I will outline what helped me...

1) Downloaded Microsoft Visual C++ 2010 SP1 Redistributable Package (x64) from the Official Microsoft Download Center page:
https://www.microsoft.com/en-us/downloa ... x?id=13523

2) Followed the instructions on this page to get at the files:
http://www.tomshardware.com/answers/id- ... 64bit.html

3a) Once you grab the set of files that were inside of the downloaded EXE, look for vc_red.cab (if you have File Extensions set to display in Windows Explorer) You can delete all other files.

3b) The tutorial on TomsHardware assumes the use of WinRAR, which I do not have. Instead, I used Universal Extractor:
http://www.lupopensuite.com/db/universalextractor.htm
(The above is an unofficial but authorized portable version in a self-extracting .EXE archive, and is my recommendation. If you'd feel safer with a download from one of the 'main' sites, use the following for the 'installer' version from Softpedia.)
http://www.softpedia.com/get/Compressio ... ctor.shtml

3c) It turns out you can also drill straight down into the .CAB file in Windows Explorer on Windows 7.

4) However you get into vc_red.cab or get files out of it, follow the rest of the instructions on the TomsHardware page: look for a file called F_CENTRAL_msvcr100_x64 and rename it to Msvcr100.dll

5) I already had a previously failed installation of Sandboxie in Program Files [not x86], so I simply copied my brand new Msvcr100.dll file into C:\Program Files\Sandboxie

6) Instant success--huzzah! I was able to start a program in Sandboxie!

**************************

I recognize that this particular issue has affected very, very few of us so far (that we know about), but Sandboxie might want to include some kind of error check in its installer for the proper Msvcr100.dll to prevent future potential users from throwing their hands up in despair/disgust and just walking away from the product. (I'm a long-time user, so...I was missing it on this PC!)
