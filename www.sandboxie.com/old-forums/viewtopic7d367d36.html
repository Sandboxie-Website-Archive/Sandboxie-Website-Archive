Post by Craig@Invincea » Thu Sep 03, 2015 11:27 am

Yuno wrote:is there some debug software or "special version" of sbie or something devs can give to us that will actually capture some data to find the problem of our broken stuff with v5?

like a dev running a test trying to replicate an issue, whatever they'd be running at the time for debugging purposes, give us a copy of whatever that dev tool is?
You can download procmon (Process Monitor) to capture that sort of stuff & save the file and post it where we can access it (files tend to be large) https://technet.microsoft.com/en-us/Lib ... 2147217396

Before running ProcMon, exit as many applications as possible to try to cut down on the system activity. Start ProcMon, no need to change any options/settings, and try to the program. As soon as it crashes, stop the ProcMon. Save the output to a PML file, be sure to select "Save All Events", zip it (important!), and upload it somewhere so we can download it.

You can repeat the same process above, with a separate PML for each, for your other issues.

Some games simply will not work. Even more probable so when you add in the changes Win 10 brings and then add the variables to each users unique systems/hardware/software that exists.

SBIE by design isolates programs (games). Thus, games when they try to install all the stuff they need to work or drivers and resources they may need to see. Can't. Now, you can play around and edit the .ini file to allow certain pipes and resource access and/or folders on the host (again, you have to know what you need to open/allow etc)

But, this totally defeats the purpose of running anything in a sandbox.
