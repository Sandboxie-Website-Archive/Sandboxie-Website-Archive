@hottroc

As for the rcpss, that can be a few things.

The "fix" was for FR Win 10 builds that. The rcpss is very rare now.

And since you mentioned a particular Rcpss instance that goes away after an attempt or 2, that's been noted before in other Beta's, and improved on.

Usually, a race timing issue and the restarting that you mention is the fix for the very few users that still have it. It has to do with how things start up and/or close within SBIE.

What happens when you "force" a program into sbie? given that this way uses a slightly different way of loading, rcpss is even more rare.

Again, it's also extremely machine dependent.

As for tweaking a "fix" for that particular error, the Dev's have done pretty much all they can do for what you describe. And the fix is either "forcing" a program (can still happen) or restart the program again (to start the "race" timing issue again)..and the odds are..It will work as that race will end differently in memory.
