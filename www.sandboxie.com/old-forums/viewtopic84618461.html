Drivers are up to date (nvidia 388.71). It doesn't matter what's launched - any sandbox, any application. Sandboxed explorer - background changing no longer works. Sandboxed GIMP - same. I was initially using DisplayFusion to change wallpapers, but after this problem initially popped up, I wiped and reinstalled Windows and have since been using the built in personalize settings app.


Repro:

*Create sandbox (no settings changed)
*Open "Run Windows Explorer" from the sandbox's menu
*Leave sandboxed window open or close it (doesn't matter after it launches)
*Right-click desktop area (non-sandboxed) and choose "Personalize"
*Select "Background" from "Personalization" submenu.
*Select Picture background and a picture.
*The background won't change.
*At this point if you uninstall Sandboxie, the background will immediately change to the selected picture (using the installer to install the same version over the currently running version has the same effect).
*Restarting the computer also causes the previously selected background to display on logon.

Hello miysis,

I was able to repro. I have made the devs aware. It seems to happen only with Windows 10 x64 FCU (did not happen with Creators Update).
To change the background make sure there are no Sandboxie processes running via Task Manager (such as Sandboxie DCOM). You should be able to switch that way, without having to uninstall or restart.

I'll update this thread once more info becomes available.
