UPDATE 2 :

After repeating the below listed steps, I can also run Firefox in the Sandbox. The first time I have to follow the 10 attempts, but after that, browsers launch normally at the first try.

These actions prove that the issue is on Norton's side, not so much on Sandboxie, since without making changes I am able to launch the applications.

I'll update this post if I find anything else.
----------------------------------------- X ---------------------------------------------------
UPDATE:

I know how this sounds but...Can somebody please test the following?
1 - Try opening IE sandboxed --> Receive the SBIE2101 message --> Click Hide.
2 - Keep trying opening IE sandboxed (almost spam it) (you may receive a sysmer.exe error during this process) .
3 - Let me know if after time 10+ IE actually starts launching Sandboxed.

I was able to repro this behavior twice (going back to my broken snapshot). After doing that, I was able open IE and Chrome Sandboxed (Firefox still returns the sbie2101 message). It could be random, but...might be worth a shot.

Another thing to try is to add the following to a desired Sandbox:
ReadFilePath=C:\Program Files\Norton Security\NortonData\22.11.0.41\Definitions\IPSDefs\20171019.001\IPSEng64.dll

----------------------------------------- X ---------------------------------------------------
All,

I was finally able to repro the issue in one VM, but several factors had to take place:
1 - Update from CU to FCU. [After this, I tested the previously installed Norton, no issues.]
2 - Have an expired Norton Security Trial.
3 - Download a new trial (this time Norton Security Deluxe (was the only one available)) .
4 - Allow Norton Security Deluxe to remove old Norton and install newer version - Reboot.
5 - Updated Sandboxie to latest beta (fully removed it, then created a new Sandbox).

After those steps, I am receiving SBIE2101 Object name not found: , error OpenProcess (C0000022) access=001FFFFF initialized=1 when I try to launch web browsers. So far, I have not been able to find a workaround but I made the devs aware of the issue.

I still need to test if re-installing Norton (fully removing it, then installing from scratch) makes a difference, since it was working fine on the previous fresh installation.

I will update this post as soon as more info becomes available.

Regards,
