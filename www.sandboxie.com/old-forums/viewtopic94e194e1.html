Windows 10 - Fall Update - [Supported in Sandboxie 5.22]

Post by Barb@Invincea » Mon Oct 16, 2017 1:07 pm

All,

Please try the latest beta 5.21.7 for full support for Windows 10 - Fall Update.

Here's a list issues that have been addressed:

- ReplaceFileW windows API function is causing a leak into the host. For example, Chrome extensions may leak outside the Sandbox and corrupt the host ones when trying to update them inside Sandboxie. [Fixed in Beta 5.21.6]

-Right-clicking on sandboxed Windows Explorer files/folders crashes explorer for 5 minutes (after 5 minutes or so, the right-click action starts working). Fixed in beta 5.21.7

-"SBIE2101 OpenProcess(C0000022) access=001FFFFF initialized=1" when using "OpenWith" via a Sandboxed Explorer --> Related to Windows Defender. Fixed in Beta 5.21.7

Please feel free to reply with any new issues you encounter while using the latest Windows Insider v17017 (Fast Ring), or once you receive the Fall Update and the latest Sandboxie beta.
Before posting a problem, be sure to test it in a new Sandbox with default settings + Sandboxie latest beta. If the issue persists, provide this information:
viewtopic.php?p=104180#p104180

For questions regarding betas (as well as download links), please see:
viewtopic.php?f=62&t=24741

Regards,
Barb.-

PS: Off-topic messages will be deleted. This thread is for windows Creator's Fall Update issues + Sandboxie latest beta ONLY.

Hi Barb, I upgraded my W10 64 bits laptop to FCU. I am glad to report that despite Sandboxie not officially supporting the Fall Creators Update as of yet, all programs and functions I run sandboxed work excellent in my laptop. I tested Firefox 32 bits, Internet explorer 11, Libre Office portable, Foxit portable, 7Zip, KMPlayer, Irfan view and Windows media player. Regarding the 2 known issues you reported in the opening post, I cant test Chrome as I dont install it in my real system but I can confirm the problem sandboxing Windows explorer. Let me point out something that might give the developers a clue for fixing the Windows explorer problem. I tested forcing my USB drives, it worked as good as ever. And lets remember, when you force USB drives, and you plug in one, the USB folder pops up open using a sandboxed version of Windows explorer. So, when forcing USB folders, Windows explorer runs sandboxed automatically with no delay. Maybe knowing that gives a ckue.
