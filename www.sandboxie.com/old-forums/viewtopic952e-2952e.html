I do not use the opera browser. After a quick search, it seems that opera stores the passwords in the "login data" file (similar to chrome).
Check if you can find the "Login Date" file the appdata directory.

So you could use:
OpenFilePath=opera.exe,%AppData%\Opera Software\*\Login Data*

The problem is not Opera. With an unsandboxed Opera, the new passwords stay. With a sandboxed Opera, on browser closure the new passwords are lost. The problem is with Sandboxie.

Opera is fine, no problems. Using Sandboxie/Opera, new passwords created are not shown in Opera when the Sandboxie/Opera session has been shut down. That means the problem is with Sandboxie not Opera. This problem is identical to the same trouble I had with lost bookmarks, recently solved in an earlier thread on this Forum.

All data in the sandbox is deleted on closure, so if Sandboxie is not configured to allow session changes like new bookmarks or passwords to leave the sandbox on closure, they are deleted and Opera does not get them.

The fix I have shown is identical to the successful bookmarks fix that solved missing new boomarks.
