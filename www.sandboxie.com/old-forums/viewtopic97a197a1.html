Mouse pointer position jumps in TeamViewer

Posted: Sat Sep 16, 2017 9:51 pm
by Scoox
Windows 10 Professional x64 Creators Update
Sandboxie v5.21
TeamViewer v12.0.86369
Not running any antivirus or firewall software (thank you Sandboxie )
Computer: Surface Pro 4
Display resolution: 2736x1824 @ 200% display scaling

I'm experiencing a problem where the mouse pointer "jumps" to a different position on the screen when hovering the remote computer display area in TeamViewer. Obviously this doesn't happen if TeamViewer is run outside the sandbox. As soon as the mouse pointer is positioned over the "hot" area (the area that controls the remote computer), the mouse pointer gets thrown to the bottom-right corner of my display (not the remote display).

I take this opportunity to mention that I've had similar problems with "jumping mouse pointers" with other less popular applications (FL Studio, NI Traktor, Bitwig, Studio One, to name a few). All these applications have one thing in common: they hide the mouse pointer. The difference between those applications and TeamViewer is that TeamViewer hides the pointer on mouse hover, whereas the other applications do so on mouse click, but the problem seems to be related.

I started experiencing this issue since I got the Surface Pro 4 so I think it may be related to high-dpi display. Setting DPI scaling to 100% in Windows Settings app didn't solve the problem.

If I remember correctly, in some applications the problem only happens when DPI scaling values not a multiple of 100% e.g. 150% or 175%.

In some applications enabling the following compatibility setting on the sandboxed .exe file solved the problem (e.g. Traktor):
2017-09-17 Compatibilitty DPI scaling.png (48.79 KiB) Viewed 615 times
Interestingly, I am able to operate these applications normally if I use the Surface Pro 4 touchscreen, both touch and surface pen.

It looks like these problem in all these different applications may share a common cause, and it might be worth investigating.

Thank you guys!
