Post by dr_jon » Mon Sep 04, 2017 4:35 am

I'm quite unused to Sandboxie misbehaving so this was odd, although it's been a while since I used it. This is with Win10 Pro x64 with all patches to date.
I checked if there was an upgrade and it installed 5.20, however looking on the forum there was nothing about it being released, but I did see comments from people using it in the 5.21 beta discussion so assumed it was okay.
Anyway I wanted to run an old version of Acrobat Reader (9.0) so decided for safety's sake to make a shiny new sandbox to do it in. I then did "Run any program" and immediately (didn't touch anything) got:
SBIE2101 Object name not found: , error OpenProcess (C0000022) access=001FFFFF initialized=1
SBIE2101 Object name not found: , error OpenProcess (C0000022) access=001FFFFF initialized=1
However changing to an existing sandbox and selecting "Run any program" was fine (as far as not getting any errors and the dialogue popped up).
Back to the new Sandbox (called PDFs and on a disk, E:, local, with 950GB free) and as soon as I click Browse in the dialogue (do not running anything yet) I get a bunch of file size limit errors for iconcaches, so I wonder why 48MB is the default for File Migration, but no matter and I change it to 300000KB.
Navigating to the reader 9.0 dir and running it gives a stream of:
SBIE2203 Failed to communicate with Sandboxie Service: *GUIPROXY_00000001 - AcroRd32.exe [C0000024]
(Well, 5 page-downs to select all after the first one and to delete them.)
I checked and the Sandboxie service is shown as running in services.

Any ideas? Would the 5.21beta be a better bet, or an older one?

Oh and P.S. Google Chrome really hates the security certificate for the Invincea.com website. (I am running an extension that will always use an https connection if it is available.)

I was able to reproduce the SBIE2203 error but not SBIE2101. In my case I was able to hide the error report without noticing any ill effects. Eg it didn't crash for me like the user reported here with DC: viewtopic.php?f=11&t=24806

I'd suggest you try simply hiding it first. If that doesn't help I also tested OpenWinClass=* and it stopped the errors from popping up but apparently there is currently a known BSOD bug with that option since 5.18 so I wouldn't recommend relying on it at this time.

Post by Barb@Invincea » Tue Sep 12, 2017 1:31 pm

Hello dr_jon,

I tested the behavior with Adobe 9 + Win 10 x64 and received the SBIE2203 message. I was able to bypass it by hiding it just like Syrinx reported. I did not receive the 2101 message.

Did you copy the settings from an existing Sandbox when creating the new one? If so, please create yet another one and make sure to select "None' under "Copy settings from existing sandbox" .

OpenWinClass=* will trigger a BSOD on WIndows 10 CU - So please do not use this option unless you are running a different OS.

Also, regarding the first error, see if you have Adobe set up as your default pdf program (click on the magnifying glasss in the Windows bar and type "choose default" , you'll get to the choose default apps by file type, make sure Adobe is listed as your default pdf app.

Regards,
