Drag & drop attachments in Thunderbird don't work when Sandboxed

Posted: Sat Apr 14, 2018 7:39 pm
by phrab
I have Thunderbird 52.7.0 & both Windows XP Sp-3 & Windows 10. I used to be able to type a message in Thunderbird, go to an Explorer window & drag a file & drop it in Thunderbird as an attachment. I can still do this IF I open Thunderbird outside of Sanboxie. However, when I open Thunderbird using Sandboxie, I get 1 of 2 behaviors:
1) If I drop the file on the message, I see the "+" sign next to my cursor, but nothing happens. There's no attachment
2) If I drop the file on the grey area where attachments are shown, the attachment box opens, but nothing is attached. Only I I then click on the attachment box, a dialog box opens, which allows me to chose a file to "open"=attach. However, I still can't drag & drop.

This didn't happen in the past (I'm not sure the last time before today that I tried attaching a document). It doesn't happen if I open Thunderbird outside of Sandboxie. If I do that, I can drag & drop files.

Is there some setting in Sandboxie that has changed this behavior, as it's happening both in Windows XP & Windows 10?
Thank you,
Phil

Hello phrab,

I can repro the behavior. Not all programs work with drag-drop, but I've found 2 workarounds for you:

Open Windows Explorer Sandboxed. That way, you can drag-drop without issues.
Use Thunderbird's File menu to attach files:
File --> Attach ---> Files(s ) worked fine Sandboxed. (I am testing on Windows 10 using Sandboxie 5.24 stable).

All Sandboxie version changes can be found here:
https://www.sandboxie.com/VersionChanges

I've made the devs aware in case they have more info to add.
