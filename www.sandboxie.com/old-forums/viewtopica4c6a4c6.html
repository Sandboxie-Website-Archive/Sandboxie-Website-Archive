Please read the readme a little more carefully -

==============
After installing the SandboxiePortable package, download Sandboxie
from the author's website (at least v3.02.25) -

http://www.sandboxie.com/index.php?DownloadSandboxie
or
http://www.sandboxie.com/phpbb/viewforum.php?f=11

Open it with 7-zip. Extract the contents of the root of the
archive
==============

Extract the executable that you downloaded from the Sandboxie website (at least version 3.02.25 or higher) using 7-zip. Nowhere did I say to install it locally. The $PLUGINSDIR is inside the executable, you'll see it when you open it with 7-zip. All file extractions should be done to your USB drive (as there shouldn't be any local install).

Post by wraithdu » Sun Nov 18, 2007 10:45 am

1 - You just need kmdutil.exe from the $PLUGINDIR directory. You shouldn't copy the whole thing.

2 - If you have a local install of v3.02.25 or higher, copy your Sandboxie.ini to the App\Sandboxie directory. Just run the launcher, and it will make the necessary changes. After that, recofigure your Sandboxie UserSettings through the program GUI (mostly hiding message windows and such, sandbox settings are retained in the sandbox sections). If you don't have an INI file from that version or higher, just run the launcher and it will copy a small default INI before it launches the program. You can configure it through the GUI as usual.

Post by wraithdu » Mon Nov 19, 2007 11:42 am

I thought about that, but the version numbering in NSIS is X.X.X.X only. That and there's no reason for me to release a new launcher for every version of Sandboxie that is released. The current launcher is actually based off the changes made in Sandboxie 3.02.25, and is simply compatible with the current higher versions.

I make new releases for new features or compatibility updates. The launcher is pretty much set at the moment until any more sensible feature requests are made, or Sandboxie changes again. The recent updates have been installer updates only, and do not necessitate current users re-download the file.

So for now there are two version numbers - the installer version, and the launcher version. The launcher version is contained in the installer file name, and the installer version is found either in the tooltip, the installer's properties, or the installer window branding text.

This is due to the domain restriction. The Portable version goes into its cleanup (Stop SbieSvc... etc) as soon as SbieCtrl.exe is stopped (ie you exit the program). If Sandboxie is run on a domain PC, SbieCtrl automatically quits, hence causing your immediate cleanup.

Sorry, but there's nothing I can do about the domain restrictions on PCs (it will work on laptops now since v3.20.01).
