Hello Axie,
(I am not native in English idiom)

I had exactly the same problem as described by you.
And the problem occurs when Web Anti Virus feature is enabled. You can check it by deactivating that in Kaspersky and try run Chrome inside Sandboxie.

But how to fix that? You should be asking yourself.

I solved this by putting a clause inside Web Antivirus feature, telling it not to check the address www.google.com.br (in my case is br, since I am from Brazil). To do that follow these steps:

1. Open Kaspersky Panel
2. Click over Settings (bottom left of the main window)
3. Click over Protection Tab
4. Find and click over Web Anti-Virus
5. A new Windows will open. Make sure to enable this feature sliding the green key at the right top of the windows
6. Click on Advanced Settings
7. Now you have to put the google address inside two links: The link Configure Exclusions and the link Configure Trusted Web Pages. But pay attention: here is a trick: you must remove http:// ou https:// from the address in order it can work. I suffered to find out that. Kaspersky seems to ignore the adress when you put the protocols before the address


I hope it can solve your issue the same way it solved to me.
