Guest10 wrote:The sandbox name is embedded in the RegHive files in the sandbox.
Hi Guest10, I haven't seen this. Although Sandboxie mounts its hives under HKEY_USERS\Sandbox_mike_DefaultBox, for example, try mounting the RegHive elsewhere:

Code: Select all

reg.exe LOAD HKLM\TempHive RegHive
For me, the mounted keys all fall under HKLM\TempHive\user and HKLM\TempHive\machine, with no reference to the sandbox name.

I regularly package my sandboxes into .zip files for later use, and the original/eventual sandbox name seems irrelevant. I think werter's suggestion is a good one.

Post by Mike » Thu Sep 23, 2010 4:44 pm

Guest10 wrote:The sandbox name will be seen on the first line that shows in Notepad.
You're absolutely right. But this line will update to whatever path you mount the hive from, and has nothing in particular to do with Sandboxie. (You can play around with your Windows user hive, NTUSER.DAT, and you'll see the same behavior.)

I'm no expert, but I would compare the stuff you see only in Notepad, such as the original registry mount point, to IP addresses in an email message header - it's interesting bookkeeping with no practical bearing on the real data. What we really care about in the hive are the keys.

I think you'll immediately see what I mean if you mount a RegHive using reg.exe or regedit - the key names start with \user or \machine and are sandbox-agnostic. They only apply to a particular sandbox while mounted as subkeys under HKU\Sandbox_mike_DefaultBox, for example.
Guest10 wrote:Which sandbox name is used when the RegHive file is mounted to the Registry?
In my experience, the hive applies to whichever sandbox directory it's located in, period. Nothing tricky here.
Guest10 wrote:Or, are the old RegHive files moved into the new sandbox along with everything else, and it still works with the old sandbox name embedded in the RegHive file?
Yes, the entire sandbox can be moved as a ready-to-run unit with all registry data preserved. This is great for packaging apps for later use, or for use on other computers.
