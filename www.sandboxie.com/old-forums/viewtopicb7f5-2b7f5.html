AVG - Web Browsers - Could not hook CoGetObject - [SOLVED]

  • Quote

Post by cafe_cyber » Sun Sep 03, 2017 9:21 am

Hi, hope someone can help me with this issue as I am desperate after trying to fix it for hours.

PROBLEM
All my programs work fine in Sandboxie except (suddenly) Internet Explorer and Chrome. When starting sandboxed browsers, I receive the following error messages:
- for Internet Explorer:
SBIE2303 Could not hook CoGetObject (1)

- for Google Chrome:
SBIE2303 Could not hook CoGetObject (1)
SBIE2303 Could not hook CoGetObject (1)
SBIE2203 Failed to communicate with Sandboxie Service: *GUIPROXY_00000001 - chrome.exe [FF000002]

CONFIG
  • Windows 7 64 bit
  • Sandboxie Version 5.20 (64 bit) registered and licence activated until 2018
  • Internet Explorer 11 without ad-blocker
  • Google Chrome Version 60.0.3112.113 (Official Build) (64-bit) without ad-blocker
  • AVG AntiVirus Free 17.6.30.29
  • Windows Firewall
  • CryptoPrevent = disabled

TRIALS AND ERRORS
None of the following tricks worked:
  • uninstalling and reinstalling Sandboxie
  • uninstalling and reinstalling Sandboxie in a different folder
  • deleting the content of the sandbox
  • creating a new sandbox
  • adding Sandboxie to the Windows firewall exception list
  • adding the Sandboxie Program Files folder to the AVG Resident Shield exceptions

MY ANALYSIS
Sandboxie was working fine before, and still works fine for all non-browsing programs (File Explorer, Audacity, VLC, etc.). Both IE and Chrome work fine when not sandboxed. So my Sandboxie install is OK, I guess. However something is blocking all sandboxed browsers from opening correctly. Is there a Sandboxie log that would show what is happening when I try to start IE or Chrome?

Post by Syrinx » Tue Sep 05, 2017 12:41 pm

For AVG, tested on 17.6.3029 along with chrome.
You can add this in the boxes you are having problems with alongside the new version of AVG to prevent the crash.

Code: Select all

ClosedFilePath=*avg*snxhk*.dll
or via the GUI Sandbox Settings > Resource Access > File Access > Blocked Access > Add >

Code: Select all

*avg*snxhk*.dll
