sub-sandbox

Posted: Fri Mar 09, 2018 11:40 am
by ZorglubSandboxed
To minimize the installation directly in windows.

I would like to suggest the possibility of creating a sandbox that serves as a software installation but care execution would be done in a sub-sandbox.

Currently, I simulate this by creating a sandbox installation and through a .bat I make a copy of the content of this sansbox in an execution sandbox and then I launch the software.

It would be nice if a similar mechanism could exist without me needing to create the commands myself.

MERGED POST #1
I would like to increase the possibility of installing drivers in a sandbox combined with the execution of software in a sub-sandbox that will inherit the visibility of the parent sansbox.

this would make clean and safe installations without polluting the windows environment.

MERGED POST #2
I would like to suggest the possibility to reuse the same sandbox with a new installation of windows.

As you know Windows being what it is ... it often happens that we have to reinstall it ... then we have to reinstall all the software!
