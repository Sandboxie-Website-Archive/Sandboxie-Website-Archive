Re: Firefox crashes after downloading files

Post by bo.elam » Wed Jan 20, 2016 4:00 pm

bjm wrote: If we could only get every user to run as Sandboxie purest.
That would be a bad idea, bjm. For someone like me, who doesn't share the computers with anyone but my wife and doesn't install any new programs in the PC, using Sandboxie on its own, works great, and its the perfect way of using it. But not everyone uses the computer like that. My computers are static, they dont change at all, and all programs that I test are either using Sandboxie or Shadow defender.

What would be great I think would be getting every Sandboxie user to run programs that are compatible with Sandboxie, doing that alone, I believe, would make most problems go away. Issues like lags, sandboxes taking a long time to delete or programs taking a while to pop up open, freezes, none of that happens if you run programs that are compatible with SBIE.

One reason I feel safer using Sandboxie on its own is that by doing so I avoid potential problems like, if I go to a site using a non restricted Start Run sandbox, and all of the sudden, the site starts downloading malkware and I start experiencing freezes like the one you describe, probably nothing would happen as all should stay in the sandbox. But who knows, maybe the conflict if there's one, between Norton and SBIE, helps the malware somehow escape the sandbox. I rather avoid that kind of possibility and that's what I am doing when I use nothing along SBIE that can water down Sandboxies protection.
