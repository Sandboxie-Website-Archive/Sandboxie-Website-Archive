Several users are reporting that Chrome 37-64 is still crashing. Unfortunately, we can't repeat the problem here.

Please try these things to see if they fix the problem
1) OpenIpcPath=\BaseNamedObjects\FontCachePort
2) Disable drop rights
3) Disable all Chrome addons
4) In Chrome advanced settings, disable hardware acceleration.
5) In Chrome advanced settings, click on "Reset browser settings"

There is a possibility that ASLR is placing our injection code into a memory location that is causing a crash (for some users). If this is the case, then rebooting might make the problem go away. For those still having problems, if you re-enable drop rights, then reboot, does the problem persist 100% of the time?
