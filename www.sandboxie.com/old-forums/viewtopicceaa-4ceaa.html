The way I did mine was to go to Options > Advanced Settings > Exceptions > Add Exception.

Exception Type - chose "Folder"

Choose File etc - Browse to Sandboxie folder (default location C:\Program Files\Sandboxie\ )

Use this exception for - (check all three) .... you probably only actually need to check "Resident Shield", possibly "Identity Protection", probably not "Manual & Scheduled Scan". I checked all to be safe but I may revisit and uncheck the latter one or two.

Be warned, this might be a complete red-herring but ....

I wonder if the variability with this problem, particularly with Win 8.1 (it seems more consistent on Win 10 from what I'm reading here) is due to differing inclusions of some Windows Updates.

There is currently a lot of bad press about some of the privacy intrusions contained in Win-10. Apparently MS is also foisting some of this on Windows 8.1 users via some Windows Update components. Because of this and discussions elsewhere, I looked for and removed certain Windows 8.1 Updates that appear to be privacy related. I can't say whether or not this has affected this issue for me. In addition there is an update (KB2919355) that, in effect, amounts to a Service Pack although MS doesn't call them that any more. This addresses many. many Windows 8.1 issues including some that occur only to some users. Possibly it would relate to this problem (or at least determine why some people see it and others don't). I do have that KB installed on my (Sandboxie now working) system. Anyone using automatic updates should have this already.

FWIW, the KB's I removed over privacy concerns are:

KB3068708
KB3022345
KB3075249
KB3080149
