Re: "Do you mean FF 64 or 32 bit in relation to sandboxie 64 or 32bit? In my case Cyberfox 32 and 64 bit crash in sandboxie 64bit.
Either.

Is it possible to install sandboxie 32bit on windows 8.1 64bit without resorting to any windows compatibility setting?
You should install the recommend version for your OS. 64 bit for 64 bit, etc.
Maybe the crash is related to Windows 8.1 64bit? Or are there sandboxie users that experience the crash on other version of windows?"
What you see here is what is reported. We address issues that have similarities or are affecting a serious amount of users.. We then look for reproducible instances. Usually, it'a one of 4 things. Not deleting the contents of the SB, An extension/Addon, Antivirus or anti-malware or something unrelated to any of those 3.
My situation is this... I discovered after seeing the dl crash problem for a long time that I had 32 and 64 bit FF installed... I think it was from doing a manual update accidentally using the standard windows installer instead of the 64 bit which I had already installed. I started to wonder if there might be a connection.
Remove all instances of FF and start again. Make sure you delete the contents of your sandbox too. Or create separate SB for 64b and 32b.
Another thing I noticed was after reapplying a Paragon image the problem seemed to clear up for while. Seeing as how the page file isnt saved in those images it would be rebuilding at that point... could there be a clue there?
Sure. We've seen issues related to that. Not directly related to SBIE, but just random "issues" that users had. (I've personally seen odd behavior in Windows and Mac OS with saved drive images and cloned drives..)
