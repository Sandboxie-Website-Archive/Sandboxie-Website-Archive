Post by Guest10 » Sun Aug 02, 2015 7:24 am

Can you run TB unsandboxed, to see if tagging works as expected?

I don't see how Sandboxie could have anything to do with this, as long as emails are successfully being saved outside of the sandbox.
When a tag is applied or unapplied to an email, Thunderbird makes a change to the "Message Source" of that email.
In order to do so, sandboxed TB must be able to edit the Message Source of an email that's outside of the sandbox.
If sandboxed TB can save (write) an email out of the sandbox, the question is: why wouldn't sandboxed TB also be able to edit the Message Source of that same email?

Select an email and then View > Message Source

The first few lines of an email without a tag should be similar to this:
From - ...
X-Account-Key: account...
X-UIDL: ALjmjkQAAA/nVa6IHwV6WIcg5x8
X-Mozilla-Status: 0001
X-Mozilla-Status2: 00000000
X-Mozilla-Keys:
X-Apparently-To: ...

So,
Without any tag:
X-Mozilla-Keys:

After an "Important" tag is applied to the email, the Message Source is changed to:
X-Mozilla-Keys: $label1

Or, after a "Work" tag is applied:
X-Mozilla-Keys: $label2

----
Add a tag to an email in the Inbox, when TB is running sandboxed.
While TB is still running sandboxed, use Windows Explorer (or "File Explorer") to look through the entire contents of the sandbox.
There should not be any file called "Inbox" anywhere inside of the sandbox, if the Message Source of that email in the Inbox was successfully changed outside of the sandbox.
