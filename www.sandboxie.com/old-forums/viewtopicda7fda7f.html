by Ruhe
The current beta adds a new feature: file hash verification.

If enabled (default setting) it stores a file hash and verifies it the next time the file is started. If the verification fails SO shows a dialog like this




Added/New settings:

Code: Select all

[Settings]
; Use session cache for 5 signed and packed/crypted files. "y" to check from
; time to time only, therefore faster. "n" to check a file every time
; for signature and/or packer/crypter, therefore more secure
UseSessionCache=y
; Store (%APPDATA%\SBObserver\Hashes.db) and verify file hash (SHA1)
CheckFileHash=y
