Unable to run Sandboxie on Windows 10

  • Quote

Post by AlphaUMi » Tue Dec 29, 2015 9:38 am

Hi! Recently, I made a "long jump", upgrading from Windows XP to Windows 10. Fresh installation from DVD, Windows 10 November Update, v1511.
Initially, Sandboxie was working well, but after a couple of days, suddenly it stopped working. The only thing I can do is open the Sandboxie Control Panel. Everything I try to launch sandboxed (explorer, a program, the browser), simply do not run, no error messages, no alert windows. What I've noticed is that the process Start.exe starts but immediately closes.

What I've tried so far:
- Several uninstallations, reboots, and reinstallations. The setup says that the Sandboxie drivers are been successfully installed and activated.
- Tried different install folders to test potential UAC conflicts, no avail.
- Tried disabling antivirus program (avast).
- Tried latest version 5.06 and even the beta version.

Please, any idea about the cause of this issue? What's wrong with my system, installed less that 1 month ago?

Post by Craig@Invincea » Tue Dec 29, 2015 12:32 pm

AlphaUMi wrote:Everything I tried is from Sandboxie Control Panel, so I tried to launch the browser from "run browser sandboxed". Firefox is my default browser. I can't even launch Explorer. Start.exe appears for a moment in the Control Panel but immediately closes. Every time I installed Sandboxie was from a fresh installation: I mean, uninstall, reboot, reinstall. There is some log I can attach?
First - and easiest to do is to remove the Antivirus product you have. Again, that's usually the #1 cause.

Second -You can try and run FF by right clicking the FF icon and selecting "run as sandboxed."

Third -You can also right click on the SBIE tray icon and select any option there to run a program.

To get a log, This will take a little work:
Get procmon here (free) https://technet.microsoft.com/en-us/sys ... 96645.aspx

Step #1
Close any programs that are not needed.
Run Procom (not sandboxed)
and then try and fire up FireFox (or the browser causing issues) (sandboxed of course).
Let the error happen/not load/etc
stop capturing events in Procmon (under file)
then Save the results (under file, make sure "save all events it ticked") and
save in PML format.
These files can be large, so post somewhere we can download and look (link)
OR (Please zip them!) and send to Support@sandboxie.com

Step #2: Also, if you can give us a snapshot of your installed software by opening Power Shell (as admin) and type:

Get-WmiObject -Class Win32_Product | Select-Object -Property Name > C:\Software\PCapps.txt

The PCapps.txt will be created listing your software, file you can send to us as well as the PML

Post by Yann » Mon Jan 04, 2016 7:03 am

UPDATE [Solution]:
After searching a little on Internet, it seems that, indeed, the problem comes from Comodo HIPS.
In order to solve this issue, it's enough to EXCLUDE the folder of Sandboxie in Comodo with: Advanced Settings>>Defense+>>HIPS>>HIPS Settings>>Detect shellcode injections (click on Exclusions then Add new rule).
