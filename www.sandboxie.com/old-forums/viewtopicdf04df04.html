I am having a problem with the Immediate Recovery function when downloading a .zip file.
I wanted to download a .zip file that I had already downloaded before. First I noticed that the Immediate Recovery window did not appear and after a while even the sandboxie control gets terminated.
After some investigation I discovered it has something to do with the file name you want to download. If there is a minus sign '-' in the file name then the Immediate Recovery window does not appear anymore.
The problem occurs when I follow these steps below:
- Download a .zip file , for example download emdb.zip from http://www.emdb.eu/
- Before saving the file change the file name to emdb-.zip
- After download complete, let sandboxie recover the file
- Download the file again and also change the file name to emdb-.zip
- After downloading you will notice that the Immediate Recovery window does not appear
- Wait for approxmitly 60 seconds
- Download the .zip file and change the file name to emdb-.zip once more and the sandboxie control is terminated.

System:
Windows 8.1 (64 bit)
Sandboxie 5.05.2 64 bit (registered)
Firefox 41.0.2

Yes, the rcpsss race issue may still happen, although it's more rare. It's more prevalent in Windows 10 and multi-core systems. This is a machine dependent error, and at this time not much can be done with that error as it has to do with hardware, parallel processing and timing. It's still being investigated, but a permant fix has not been found.

If you can post your hardware here and the software you are running by running this command in powershell:
Get-WmiObject -Class Win32_Product | Select-Object -Property Name > C:\Software\PCapps.txt

The pcapps.txt is the name of the file it will create. you can email that to support@sandboxie.com
