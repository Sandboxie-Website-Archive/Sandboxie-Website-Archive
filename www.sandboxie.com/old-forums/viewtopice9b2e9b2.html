by tzuk
Yes, I can elaborate, but I have only looked into this briefly a couple of months ago, so please keep in mind that this is just first impressions of Windows 8.

Earlier versions of Windows/IE store temporary internet files (cache) and history information as standalone files, so there is no issue with IE in the sandbox accessing this information alongside other programs (like IE outside the sandbox).

In Windows 8 and IE 10 there is a caching service that manages all this information in an opaque database which is accessible only to that service. So if you start an instance of IE in the sandbox, it has to talk to an instance of this caching service as well.

One option is the default behavior where a copy of this caching service is started in the sandbox (inside a "dllhost" program), but it has an empty database because the database outside the sandbox is not accessible to make a copy of it.

The other option is to apply some settings* that would let the sandboxed IE talk to the caching service outside the sandbox, but that means cache and history information is exchanged both ways between IE and the caching service.

* I have to point out that I don't think I have those settings at this time, and that I plan to revisit this issue later on to see if there is anything I can do about it.
