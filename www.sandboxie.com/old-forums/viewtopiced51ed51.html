But I suspect it all started with 5.12 because of “The new, faster loader code developed for Windows 10 is now used for Win 8.0 & 8.1". Sorry I did not indicate before that I run Win 8.1.
Doubtful.
It is also my rule of thumb not to install older programs directly into a modern OS,
SBIE is not guaranteed to be able to host and run any and all programs. You'd need a VM for that. You need to realize the difference between an isolated container and a VM.
A VM is heavy, because it needs to be able to host a full fledged OS and the programs along with (drivers, services, etc.) which is NOT possible in SBIE.
A container is simply a layer over your OS.
I thought it would be of interest for developers to know about that unexpected effect.
Thanks, but you need to know the limits of a container vs a an actual VM. SBIE was never designed to be a VM or to host any and all programs. Simply impossible.

Sanboxie was designed to run IE in a container. Hence, SandboxIE. That's the core of it, that's what it does, but now we have many many browsers, office suites, chat clients and some programs that can be installed directly into the SB (most browsers) and almost ALL programs can be "run as sandboxed" when installed on your host.
