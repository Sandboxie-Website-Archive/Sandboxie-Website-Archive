Indexing in Outlook 2013 (Click-to-Run)

Posted: Sat Oct 28, 2017 5:00 pm
by jclarkw
When running sandboxed, Outlook does not seem to update my index of messages when I get new messages or move them to different folders. Maybe I just noticed this, or maybe something has changed. I'm running Outlook 2013 offline (all e-mail accounts are POP3) from a click-to-run installation of Office under Windows 7 SP1 and Sandboxie 5.20 64-bit.

Example: I just forced Windows to rebuild its index, which includes both Outlook itself (for my user) and the directory where the Outlook data file (.PST) resides. When Outlook was then opened inside the sandbox, search could not find anything at all! When I opened Outlook outside the sandbox and searched, however, it said it was indexing and did start to find things for me. Now that Outlook's indexing appears to have completed, if I close it and then re-open inside the sandbox, search appears to work properly.

Although I'm delighted that it is now possible to run click-to-run Office applications sandboxed, I'm wondering if I need to permit sandboxed Outlook to make changes to the Windows index that lives somewhere outside the sandbox. Or is there some other problem at work here?
