i normally run with opera as my default browser and i force it to always run sandboxed by setting the appropriate options in sandboxie. this allows me to click links in emails and miscellaneous web pages and have them always open in a sandboxed browser. i have been operating this way for years.

upon returning from a 4 day trip on 7/14/13, clicking links in emails results in search results displayed in japanese [?] characters. the computer was hibernated and the power shut off at the UPS while we were gone.

e.g.
clicking http://www.iheart.com/ results in

http://www.google.com/search?client=opera&q=%E6%A0%.......

the google web page says:
"脂" (and any subsequent words) was ignored because we limit queries to 32 words.

if i rt clk the link and select 'copy link location' and then paste into the browser it goes to the right web page.

if i remove the forced sandbox from opera, link clicking works properly.

if i make firefox my default browser, link clicking works ok whether or not FF is forced to sandbox.

i use FF for most of my browsing and thus don't want to run it forced sandbox mode.

parameters:
win7pro 64
sandboxie 4.04 64bit
opera 12.16
firefox ESR 17.0.8

help!

Post by tzuk » Wed Aug 28, 2013 1:38 pm

Obscure, yes. Interesting, no. This was a simple matter of a conversion error between ANSI and UNICODE text strings.
Opera sends the link text as a single byte string (ANSI) and Sandboxie thought it was supposed to be a UNICODE string.
