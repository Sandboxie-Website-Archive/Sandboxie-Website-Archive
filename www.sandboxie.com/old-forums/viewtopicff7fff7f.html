Ace Online in Sandboxie causes BSOD after Win 10 Creators Update

Posted: Thu Sep 21, 2017 4:36 pm
by Kevin87
Hello all,

I wish to report an issue with the game "Ace Online", hosted by Subagames. It ran well in Sandboxie 5.20 in Windows 10 before the Creators Update.

Suddenly, after upgrading to the Windows 10 Creators Update, Windows crashes with a bsod when Ace Online is started within Sandboxie. This happens on all pc's and laptops I own that run the Windows 10 Creators Update. It doesn't seem to matter if it's Windows 10 Home or Pro.

What additional information do you require? Maybe it’s an idea to check out the game yourself on https://ace.subagames.com/index.aspx

In case you wish to try the game yourself to figure out what is causing this issue, some steps need to be performed when creating a sandbox to run the game. These settings are confirmed to work in Windows 7 and Windows 10 (before the Creators update):

Create a new sandbox – Copy settings from “none”
Set the following sandbox settings:
  • Restrictions- drop rights- checkbox has to be unchecked.
  • Resource access – file access – full access – add – c:\Suba Games\ - apply. (or whatever location you install the game to)
  • Resource access – window access – add - * - ok – apply

Note: Once you are logged in into the game, the game (whether in Sandboxie or not) can crash if admin rights are asked by another program. This is independent on the usage of Sandboxie or any version of Windows and does not happen all the time. I assume this is completely unrelated.

If you require more information, I'm happy to provide this.
